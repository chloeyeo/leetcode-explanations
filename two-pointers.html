#167
Two Sum II - Input Array Is Sorted
🔍 High Level
정렬된 배열에서 두 수의 합이 target이 되는 인덱스(1-based)를 찾는다. 정렬되어 있으므로 양 끝에서 좁혀가는 Two Pointers로 O(n)에 해결.
numbers = [2, 7, 11, 15], target = 9 idx: 0 1 2 3 val: [2, 7, 11, 15] L R sum = 2+15 = 17 > 9 → R-- L R sum = 2+11 = 13 > 9 → R-- L R sum = 2+7 = 9 ✅ return [1, 2] 규칙: sum < target → L++ (더 큰 값 필요) sum > target → R-- (더 작은 값 필요) sum == target → 정답
💻 기본 풀이
각 원소에 대해 (target - num)을 이진 탐색으로 찾는 방법. O(n log n).
🐍 Python
def twoSum(numbers, target):
    from bisect import bisect_left
    for i in range(len(numbers)):
        complement = target - numbers[i]
        j = bisect_left(numbers, complement, i + 1)
        if j < len(numbers) and numbers[j] == complement:
            return [i + 1, j + 1]
⚡ 최적화 버전 OPTIMAL
양 끝에서 시작하는 Two Pointers. 정렬된 성질을 이용해 한 번의 스캔으로 해결 → O(n).
🐍 Python
def twoSum(numbers, target):
    left, right = 0, len(numbers) - 1
    while left < right:
        s = numbers[left] + numbers[right]
        if s == target:
            return [left + 1, right + 1]
        elif s < target:
            left += 1
        else:
            right -= 1
📊 복잡도 분석
left, right = 0, n-1
양 끝 포인터 초기화 → O(1)

두 포인터가 만날 때까지 → 최대 n번
s = numbers[left] + numbers[right]
두 값 합 → O(1)
left += 1 / right -= 1
매 반복마다 한 포인터 이동 → O(1)
⏱ 시간
O(n)
두 포인터가 한 번에 한 칸씩 이동
💾 공간
O(1)
포인터 2개만 사용

#15
3Sum
🔍 High Level
배열에서 합이 0인 세 수의 조합을 중복 없이 모두 찾는다. 정렬 후 각 i를 고정하고 (i+1, n-1) 범위에서 Two Pointers로 나머지 두 수를 탐색.
nums = [-1, 0, 1, 2, -1, -4] 정렬: [-4, -1, -1, 0, 1, 2] i=0 (-4): L=1, R=5 → -4 + -1 + 2 = -3 → L++ -4 + 0 + 2 = -2 → L++ ... ✗ i=1 (-1): L=2, R=5 → -1 + -1 + 2 = 0 ✅ [-1,-1,2] L=3, R=4 → -1 + 0 + 1 = 0 ✅ [-1,0,1] i=2 (-1): 직전과 같음 → skip (중복 제거) i=3 ( 0): L=4, R=5 → 0 + 1 + 2 = 3 → R-- ✗ 핵심: i > 0 이면 break (모두 양수) 중복은 nums[i] == nums[i-1] 일 때 skip
💻 기본 풀이
삼중 루프 + set으로 중복 제거. O(n³) 시간 — 큰 입력에서 TLE.
🐍 Python
def threeSum(nums):
    nums.sort()
    res = set()
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    res.add((nums[i], nums[j], nums[k]))
    return [list(t) for t in res]
⚡ 최적화 버전 OPTIMAL
정렬 + Two Pointers로 O(n²)에 해결. 양수 구간 진입 시 조기 종료, 동일값 skip으로 중복 제거.
🐍 Python
def threeSum(nums):
    nums.sort()
    res = []
    n = len(nums)
    for i in range(n - 2):
        if nums[i] > 0: break
        if i > 0 and nums[i] == nums[i - 1]: continue
        left, right = i + 1, n - 1
        while left < right:
            s = nums[i] + nums[left] + nums[right]
            if s == 0:
                res.append([nums[i], nums[left], nums[right]])
                left += 1; right -= 1
                while left < right and nums[left] == nums[left - 1]: left += 1
                while left < right and nums[right] == nums[right + 1]: right -= 1
            elif s < 0: left += 1
            else: right -= 1
    return res

기준 원소 i 고정 → n번

각 i마다 Two Pointers 스캔 → O(n)
if nums[i] > 0: break
양수 진입 시 조기 종료 (가지치기)
nums[i] == nums[i-1]: continue
동일값 skip → 중복 조합 방지
⏱ 시간
O(n²)
바깥 루프 n × Two Pointers n
💾 공간
O(1)

#42
Trapping Rain Water
🔍 High Level
높이 배열에서 비가 내릴 때 고이는 물의 총량. 각 칸의 물 = min(왼쪽 최대, 오른쪽 최대) - 현재 높이. Two Pointers + leftMax/rightMax 트래킹으로 O(1) 공간에 해결.
height = [0,1,0,2,1,0,1,3,2,1,2,1] █ █ ░ ░ ░ █ █ ░ █ █ █ ░ █ █ █ █ █ █ ───────────────────── 0 1 0 2 1 0 1 3 2 1 2 1 각 칸의 물 = min(leftMax, rightMax) - height[i] 단, 음수면 0 Two Pointers 아이디어: height[L] < height[R] 이면 → 왼쪽이 병목 leftMax가 R쪽 어느 값보다 작음이 보장됨 → water[L] = leftMax - height[L], L++ 그 반대면 → 오른쪽에서 처리, R-- 총 물량 = 6
💻 기본 풀이
각 인덱스 i에 대해 leftMax[i], rightMax[i]를 미리 계산. 직관적이지만 O(n) 추가 공간 사용.
🐍 Python
def trap(height):
    n = len(height)
    if n == 0: return 0
    leftMax = [0] * n
    rightMax = [0] * n
    leftMax[0] = height[0]
    for i in range(1, n):
        leftMax[i] = max(leftMax[i - 1], height[i])
    rightMax[n - 1] = height[n - 1]
    for i in range(n - 2, -1, -1):
        rightMax[i] = max(rightMax[i + 1], height[i])
    return sum(min(leftMax[i], rightMax[i]) - height[i] for i in range(n))
⚡ 최적화 버전 OPTIMAL
Two Pointers로 leftMax/rightMax를 변수로만 유지 → 공간 O(1). 낮은 쪽에서 항상 물량이 결정되는 성질 활용.
🐍 Python
def trap(height):
    left, right = 0, len(height) - 1
    leftMax = rightMax = water = 0
    while left < right:
        if height[left] < height[right]:
            if height[left] >= leftMax:
                leftMax = height[left]
            else:
                water += leftMax - height[left]
            left += 1
        else:
            if height[right] >= rightMax:
                rightMax = height[right]
            else:
                water += rightMax - height[right]
            right -= 1
    return water

O(n)
