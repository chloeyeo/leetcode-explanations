LeetCode #206 — Reverse Linked List
Easy
🎨 Algorithm Visualization
Input: 1 → 2 → 3 → 4 → 5 → None Step 1: prev=None cur=1 None ← 1 2 → 3 → 4 → 5 Step 2: prev=1 cur=2 None ← 1 ← 2 3 → 4 → 5 Step 3: prev=2 cur=3 None ← 1 ← 2 ← 3 4 → 5 Step 4: prev=3 cur=4 None ← 1 ← 2 ← 3 ← 4 5 Step 5: prev=4 cur=5 None ← 1 ← 2 ← 3 ← 4 ← 5 Output: 5 → 4 → 3 → 2 → 1 → None
🐍 Basic Solution
# Definition for singly-linked list. class ListNode: def __init__(self, val=0, next=None): self.val = val self.next = next class Solution: def reverseList(self, head: ListNode) -> ListNode: prev = None cur = head while cur: next_node = cur.next # save next cur.next = prev # reverse pointer prev = cur # move prev forward cur = next_node # move cur forward return prev
⚡ Optimized Solution (Recursive)
class Solution: def reverseList(self, head: ListNode) -> ListNode: if not head or not head.next: return head new_head = self.reverseList(head.next) head.next.next = head # reverse the link head.next = None # cut old forward link return new_head
📊 Complexity Analysis
VersionTimeSpaceReason
IterativeO(n)O(1)Single pass; only 3 pointers used
RecursiveO(n)O(n)n recursive calls on the call stack
LeetCode #142 — Linked List Cycle II
Medium
🎨 Algorithm Visualization (Floyd's Tortoise & Hare)
List: 3 → 1 → 2 → 4 ↑_____| (cycle starts at node 2) Phase 1 — Detect cycle: slow: 3→1→2→4→2 (moves +1 each step) fast: 3→2→2→4→2 (moves +2 each step) MEET at node 2 (or 4) Phase 2 — Find entry: ptr1 = head (3) ptr2 = meeting point Move both +1 until they meet: ptr1: 3→1→2 ptr2: meet→4→2 ENTRY = node 2 ✓
🐍 Basic Solution (Hash Set)
class Solution: def detectCycle(self, head: ListNode) -> ListNode: visited = set() cur = head while cur: if cur in visited: return cur # cycle entry found visited.add(cur) cur = cur.next return None # no cycle
⚡ Optimized Solution (Floyd's Algorithm)
class Solution: def detectCycle(self, head: ListNode) -> ListNode: slow = fast = head # Phase 1: detect meeting point while fast and fast.next: slow = slow.next fast = fast.next.next if slow == fast: # Phase 2: find cycle entry ptr = head while ptr != slow: ptr = ptr.next slow = slow.next return ptr return None
📊 Complexity Analysis
VersionTimeSpaceReason
Hash SetO(n)O(n)Stores every visited node in a set
Floyd'sO(n)O(1)Two-pointer math; no extra storage needed
LeetCode #25 — Reverse Nodes in k-Group
Hard
🎨 Algorithm Visualization (k=2)
Input: 1 → 2 → 3 → 4 → 5, k=2 Group 1: [1,2] → reverse → 2→1 Group 2: [3,4] → reverse → 4→3 Group 3: [5] → k nodes? NO → keep as-is Result: 2 → 1 → 4 → 3 → 5 Pointer trace: dummy → 1 → 2 → 3 → 4 → 5 groupPrev = dummy Step A: count k nodes ahead → tail=2 Step B: save next group start = 3 Step C: reverse [1→2] to [2→1] Step D: connect: dummy→2→1→3... Step E: groupPrev moves to node 1 Repeat for next group...
🐍 Basic Solution (Extra Array)
class Solution: def reverseKGroup(self, head: ListNode, k: int) -> ListNode: nodes = [] cur = head while cur: nodes.append(cur) cur = cur.next # Reverse every k-chunk in place for i in range(0, len(nodes) - k + 1, k): nodes[i:i+k] = nodes[i:i+k][::-1] # Reconnect nodes for i in range(len(nodes) - 1): nodes[i].next = nodes[i+1] nodes[-1].next = None return nodes[0]
⚡ Optimized Solution (In-place Pointer Reversal)
class Solution: def reverseKGroup(self, head: ListNode, k: int) -> ListNode: dummy = ListNode(0) dummy.next = head group_prev = dummy while True: # Check if k nodes remain kth = self.get_kth(group_prev, k) if not kth: break group_next = kth.next # Reverse k nodes prev, cur = kth.next, group_prev.next while cur != group_next: nxt = cur.next cur.next = prev prev = cur cur = nxt # Reconnect group tmp = group_prev.next group_prev.next = kth group_prev = tmp return dummy.next def get_kth(self, curr: ListNode, k: int) -> ListNode: while curr and k > 0: curr = curr.next k -= 1 return curr

VersionTimeSpaceReason
Array-basedO(n)O(n)Stores all nodes in list; O(n) slice ops
In-placeO(n)O(1)Each node visited once; only pointer vars used
