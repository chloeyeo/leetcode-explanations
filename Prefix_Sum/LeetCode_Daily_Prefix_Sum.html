LeetCode #303  Range Sum Query - Immutable Easy — by LeetCode

Algorithm Visualization

nums = [-2, 0, 3, -5, 2, -1] index = 0 1 2 3 4 5 Build prefix array: pre[0] = 0 pre[1] = -2 pre[2] = -2+0 = -2 pre[3] = -2+3 = 1 pre[4] = 1-5 = -4 pre[5] = -4+2 = -2 pre[6] = -2-1 = -3 sumRange(0,2) = pre[3]-pre[0] = 1-0 = 1 OK sumRange(2,5) = pre[6]-pre[2] = -3-(-2)= -3 OK

Basic Solution

class NumArray: def __init__(self, nums): self.nums = nums # O(1) init def sumRange(self, left, right): return sum(self.nums[left:right+1]) # O(n) query

Optimized Solution

class NumArray: def __init__(self, nums): self.pre = [0] * (len(nums)+1) for i,v in enumerate(nums): self.pre[i+1] = self.pre[i]+v # build once O(n) def sumRange(self, left, right): return self.pre[right+1]-self.pre[left] # O(1)

LeetCode #560  Subarray Sum Equals K Medium — by LeetCode

Algorithm Visualization

nums=[1,1,1], k=2 init: {0:1} idx=0 pre=1 {0:1,1:1} found(1-2=-1)? No idx=1 pre=2 {0:1,1:2,2:1} found(2-2=0)? Yes count=1 idx=2 pre=3 {0:1,1:2,2:1,3:1} found(3-2=1)? Yes count=3 Answer: 3

Basic Solution

def subarraySum(nums, k): count = 0 for i in range(len(nums)): total = 0 for j in range(i, len(nums)): total += nums[j] if total == k: count += 1 return count # O(n^2)

Optimized Solution

from collections import defaultdict def subarraySum(nums, k): pc = defaultdict(int); pc[0] = 1 pre = count = 0 for num in nums: pre += num count += pc[pre - k] # subarrays ending here pc[pre] += 1 return count # O(n)

Complexity Analysis

OperationBasicOptimizedReason
TimeO(n²)O(n)Hash map O(1) lookup per element
SpaceO(1)O(n)Hash map stores prefix sums
Key insightprefix[j]-prefix[i]=k means subarray i+1..j sums to k
LeetCode #329  Longest Increasing Path in a Matrix Hard — by LeetCode

Algorithm Visualization

matrix=[[9,9,4],[6,6,8],[2,1,1]] DFS+Memoization from each cell: (2,1)=1->(2,0)=2->(1,0)=6->(0,0)=9 len=4 memo[r][c] = longest increasing path from (r,c) memo table: [1,1,1] / [4,1,1] / [3,2,1] Best = 4

Basic Solution

def longestIncreasingPath(matrix): if not matrix: return 0 R,C = len(matrix),len(matrix[0]) dirs = [(0,1),(0,-1),(1,0),(-1,0)] def dfs(r,c): best = 1 for dr,dc in dirs: nr,nc = r+dr,c+dc if 0<=nr<R and 0<=nc<C and matrix[nr][nc]>matrix[r][c]: best = max(best,1+dfs(nr,nc)) return best return max(dfs(r,c) for r in range(R) for c in range(C))

Optimized Solution

from functools import lru_cache def longestIncreasingPath(matrix): if not matrix: return 0 R,C = len(matrix),len(matrix[0]) dirs = [(0,1),(0,-1),(1,0),(-1,0)] @lru_cache(maxsize=None) def dfs(r,c): best = 1 for dr,dc in dirs: nr,nc = r+dr,c+dc if 0<=nr<R and 0<=nc<C and matrix[nr][nc]>matrix[r][c]: best = max(best,1+dfs(nr,nc)) # cached return best return max(dfs(r,c) for r in range(R) for c in range(C))

Complexity Analysis

OperationBasicOptimizedReason
TimeO(2^(R*C))O(R*C)Each cell computed once with lru_cache
SpaceO(R*C)O(R*C)Call stack + memo table
Key insightStrictly increasing = DAG, no cycles, safe to memoize
