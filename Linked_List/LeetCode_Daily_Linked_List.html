LeetCode #206
Reverse Linked List
by LeetCode
📊 알고리즘 시각화
Input: 1 -> 2 -> 3 -> 4 -> 5 -> None Step 1: prev=None curr=1 None <- 1 2 -> 3 -> 4 -> 5 Step 2: prev=1 curr=2 None <- 1 <- 2 3 -> 4 -> 5 Step 3: prev=2 curr=3 None <- 1 <- 2 <- 3 4 -> 5 Step 4: prev=3 curr=4 None <- 1 <- 2 <- 3 <- 4 5 Step 5: prev=4 curr=5 None <- 1 <- 2 <- 3 <- 4 <- 5 Output: 5 -> 4 -> 3 -> 2 -> 1 -> None
💻 기본 풀이 (Iterative)
class ListNode: def __init__(self, val=0, next=None): self.val = val self.next = next class Solution: def reverseList(self, head): prev = None curr = head while curr: nxt = curr.next # 다음 노드 저장 curr.next = prev # 방향 반전 prev = curr # prev 전진 curr = nxt # curr 전진 return prev
⚡ 최적화 버전 (Recursive)
✦ Recursive
class Solution: def reverseList(self, head): if not head or not head.next: return head new_head = self.reverseList(head.next) head.next.next = head head.next = None return new_head
📈 복잡도 분석
버전시간공간이유
IterativeO(n)O(1)노드 1회 순회 / 포인터 3개만 사용
RecursiveO(n)O(n)노드 1회 순회 / 콜 스택 n 레벨

LeetCode #19
Remove Nth Node From End of List
by LeetCode
📊 알고리즘 시각화
Input: 1 -> 2 -> 3 -> 4 -> 5, n = 2 [1단계] dummy 추가 + fast를 n+1=3 칸 전진 slow=dummy fast=node(3) [2단계] slow, fast 동시 이동 (fast가 None까지) slow=dummy fast=node(3) slow=node(1) fast=node(4) slow=node(2) fast=node(5) slow=node(3) fast=None -> STOP [3단계] slow.next(=node(4)) 제거 slow.next = slow.next.next Result: 1 -> 2 -> 3 -> 5
💻 기본 풀이 (Two Pass)
class Solution: def removeNthFromEnd(self, head, n): length = 0 curr = head while curr: length += 1 curr = curr.next dummy = ListNode(0) dummy.next = head curr = dummy for _ in range(length - n): curr = curr.next curr.next = curr.next.next return dummy.next
⚡ 최적화 버전 (One Pass Two Pointers)
✦ One Pass
class Solution: def removeNthFromEnd(self, head, n): dummy = ListNode(0) dummy.next = head fast = slow = dummy for _ in range(n + 1): fast = fast.next while fast: fast = fast.next slow = slow.next slow.next = slow.next.next return dummy.next
📈 복잡도 분석
버전시간공간이유
Two PassO(n)O(1)길이 계산 + 타겟 탐색 (2번 순회)
One PassO(n)O(1)fast/slow 포인터로 한 번에 처리

LeetCode #25
Reverse Nodes in k-Group
by LeetCode
📊 알고리즘 시각화
Input: 1 -> 2 -> 3 -> 4 -> 5, k = 2 [그룹1] k=2개 확인: 1->2 존재 v 바꾸기 전: prev -> 1 -> 2 -> 3 -> 4 -> 5 바꾸기 후: prev -> 2 -> 1 -> 3 -> 4 -> 5 [그룹2] k=2개 확인: 3->4 존재 v 바꾸기 전: 1(prev) -> 3 -> 4 -> 5 바꾸기 후: 1(prev) -> 4 -> 3 -> 5 [그룹3] 5만 남음 -> 2개 미만 x 그대로 Output: 2 -> 1 -> 4 -> 3 -> 5
💻 기본 풀이 (Stack 활용)
class Solution: def reverseKGroup(self, head, k): dummy = ListNode(0) dummy.next = head prev = dummy while True: check = prev for _ in range(k): check = check.next if not check: return dummy.next stack, curr = [], prev.next for _ in range(k): stack.append(curr.val) curr = curr.next curr = prev.next while stack: curr.val = stack.pop() curr = curr.next for _ in range(k): prev = prev.next return dummy.next
⚡ 최적화 버전 (In-place Pointer Reversal)
✦ In-Place O(1) Space
class Solution: def reverseKGroup(self, head, k): def get_kth(node, k): while node and k > 0: node = node.next; k -= 1 return node dummy = ListNode(0) dummy.next = head prev = dummy while True: kth = get_kth(prev, k) if not kth: break group_next = kth.next p, c = None, prev.next while c != group_next: nxt = c.next c.next = p p = c; c = nxt prev.next.next = group_next tmp = prev.next prev.next = kth prev = tmp return dummy.next
📈 복잡도 분석
버전시간공간이유
StackO(n)O(k)전체 순회 / 그룹당 스택 k 크기
In-placeO(n)O(1)포인터 조작만으로 반전, 추가 메모리 없음
