LeetCode #200 — Number of Islands
Easy by LeetCode
📐 Algorithm Visualization
Grid BFS/DFS — Flood Fill Grid: 1 1 0 0 0 Step 1: Find '1' at (0,0) → DFS flood fill 1 1 0 0 0 Step 2: Mark visited cells as '0' 0 0 1 0 0 Step 3: Count = 1 0 0 0 1 1 Step 4: Find next '1' at (2,2) → Count = 2 Step 5: Find next '1' at (3,3) → Count = 3 DFS from (r,c): (r,c)='1' ──► mark '0' ──► recurse 4 neighbors ↙ ↘ (r-1,c) (r+1,c) (r,c-1) (r,c+1) Result: 3 islands
💻 Basic Solution (DFS)
class Solution: def numIslands(self, grid: list[list[str]]) -> int: if not grid: return 0 rows, cols = len(grid), len(grid[0]) count = 0 def dfs(r, c): # Base case: out of bounds or water/visited if r < 0 or r >= rows or c < 0 or c >= cols: return if grid[r][c] != '1': return # Mark as visited by sinking the island grid[r][c] = '0' # Explore all 4 directions dfs(r + 1, c) dfs(r - 1, c) dfs(r, c + 1) dfs(r, c - 1) for r in range(rows): for c in range(cols): if grid[r][c] == '1': # Found new island, flood fill it dfs(r, c) count += 1 return count
⚡ Optimized Solution (BFS Iterative)
from collections import deque class Solution: def numIslands(self, grid: list[list[str]]) -> int: if not grid: return 0 rows, cols = len(grid), len(grid[0]) count = 0 for r in range(rows): for c in range(cols): if grid[r][c] == '1': count += 1 grid[r][c] = '0' # Mark visited immediately queue = deque([(r, c)]) while queue: row, col = queue.popleft() for dr, dc in [(1,0), (-1,0), (0,1), (0,-1)]: nr, nc = row + dr, col + dc if (0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == '1'): grid[nr][nc] = '0' queue.append((nr, nc)) return count
📊 Complexity Analysis
VersionTimeSpaceReason
Basic DFSO(M×N)O(M×N)Visit each cell once; recursion stack up to M×N deep
Optimized BFSO(M×N)O(min(M,N))Same time; BFS queue holds at most min(M,N) cells (diagonal wavefront)
outer loopO(M×N)—Iterates all cells; already-visited cells skip instantly (O(1))
BFS innerO(k)O(k)k = island size; each neighbor enqueued at most once
LeetCode #207 — Course Schedule
Medium by LeetCode
📐 Algorithm Visualization
Cycle Detection in Directed Graph (DFS with 3-color marking) numCourses=4, prerequisites=[[1,0],[2,0],[3,1],[3,2]] Graph: 0 ──► 1 ──► 3 │ ▲ └──► 2 ──────┘ 3-Color DFS States: WHITE (0) = unvisited GRAY (1) = in current DFS path ← cycle if we revisit! BLACK (2) = fully processed (safe) DFS(0): 0ₒGRAY → DFS(1)→GRAY → DFS(3)→GRAY → done→BLACK ↑ DFS(2)→GRAY→3 already BLACK ✓ No cycle found → return True ✓
💻 Basic Solution (DFS Cycle Detection)
class Solution: def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool: # Build adjacency list graph = [[] for _ in range(numCourses)] for course, prereq in prerequisites: graph[prereq].append(course) # 0=unvisited, 1=visiting(gray), 2=visited(black) state = [0] * numCourses def has_cycle(node): if state[node] == 1: # Back edge → cycle! return True if state[node] == 2: # Already fully explored return False state[node] = 1 # Mark as being visited for neighbor in graph[node]: if has_cycle(neighbor): return True state[node] = 2 # Mark as fully processed return False # Check all nodes (graph may be disconnected) for i in range(numCourses): if has_cycle(i): return False return True
⚡ Optimized Solution (Kahn's Algorithm / Topological Sort BFS)
from collections import deque class Solution: def canFinish(self, numCourses: int, prerequisites: list[list[int]]) -> bool: graph = [[] for _ in range(numCourses)] indegree = [0] * numCourses for course, prereq in prerequisites: graph[prereq].append(course) indegree[course] += 1 # Start with all courses that have no prerequisites queue = deque(i for i in range(numCourses) if indegree[i] == 0) completed = 0 while queue: course = queue.popleft() completed += 1 for next_course in graph[course]: indegree[next_course] -= 1 if indegree[next_course] == 0: queue.append(next_course) # If we completed all courses, no cycle exists return completed == numCourses
📊 Complexity Analysis
VersionTimeSpaceReason

Kahn's BFSO(V+E)O(V+E)Build graph O(E), process each node/edge once O(V+E)
indegree updateO(E)O(V)Each edge decrements indegree exactly once
queue opsO(V)O(V)Each vertex enqueued/dequeued at most once
LeetCode #127 — Word Ladder
Hard by LeetCode
📐 Algorithm Visualization
Bidirectional BFS — Shortest Path in Word Graph beginWord="hit", endWord="cog" wordList=["hot","dot","dog","lot","log","cog"] Each edge = 1 letter change: hit ──► hot ──► dot ──► dog ──► cog (length=5) └──► lot ──► log ──► cog Bidirectional BFS meets in the middle: Forward (from "hit"): {hit} → {hot} → {dot,lot} Backward (from "cog"): {cog} → {dog,log} → {dot,lot} ← MEET! Pattern trick: "hot" → "h*t", "*ot", "ho*" Build pattern dict: {"*ot": ["hot","dot","lot"], ...} Answer: 5
💻 Basic Solution (BFS)
from collections import deque, defaultdict class Solution: def ladderLength(self, beginWord: str, endWord: str, wordList: list[str]) -> int: word_set = set(wordList) if endWord not in word_set: return 0 # Preprocess: build pattern → words map pattern_map = defaultdict(list) for word in wordList: for i in range(len(word)): pattern = word[:i] + '*' + word[i+1:] pattern_map[pattern].append(word) queue = deque([(beginWord, 1)]) visited = {beginWord} while queue: word, length = queue.popleft() for i in range(len(word)): pattern = word[:i] + '*' + word[i+1:] for neighbor in pattern_map[pattern]: if neighbor == endWord: return length + 1 if neighbor not in visited: visited.add(neighbor) queue.append((neighbor, length + 1)) return 0
⚡ Optimized Solution (Bidirectional BFS)
from collections import defaultdict class Solution: def ladderLength(self, beginWord: str, endWord: str, wordList: list[str]) -> int: word_set = set(wordList) if endWord not in word_set: return 0 pattern_map = defaultdict(list) for word in wordList: for i in range(len(word)): pattern_map[word[:i] + '*' + word[i+1:]].append(word) # Two frontiers expanding toward each other front = {beginWord: 1} back = {endWord: 1} visited_f = {beginWord} visited_b = {endWord} def expand(current, other, visited_cur, visited_oth): next_level = {} for word in current: steps = current[word] for i in range(len(word)): pattern = word[:i] + '*' + word[i+1:] for neighbor in pattern_map[pattern]: if neighbor in other: return steps + other[neighbor] if neighbor not in visited_cur: visited_cur.add(neighbor) next_level[neighbor] = steps + 1 return next_level while front and back: # Always expand the smaller frontier if len(front) <= len(back): result = expand(front, back, visited_f, visited_b) else: result = expand(back, front, visited_b, visited_f) if isinstance(result, int): return result if len(front) <= len(back): front = result else: back = result return 0
📊 Complexity Analysis
VersionTimeSpaceReason
Basic BFSO(M²×N)O(M²×N)M=word length, N=wordList size; M patterns per word, each pattern has ≄N words
Bidirectional BFSO(M²×N)O(M²×N)Same asymptotic but explores ~√(search space) in practice (meets in middle)
pattern_map buildO(M×N)O(M×N)N words × M patterns each = M×N entries
BFS per levelO(M×branching)O(level width)Bidirectional halves the depth → exponential speedup in dense graphs
