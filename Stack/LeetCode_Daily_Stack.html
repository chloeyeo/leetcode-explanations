LeetCode #20 — Valid Parentheses Easy by LeetCode
🎨 알고리즘 시각화
Input: s = "({[]})" Push/Pop 진행: ┌───────────────────────────────────────┐ │ char │ action │ stack (top→) │ ├────────────────────────────────────────┤ │ ( │ push '(' │ [ ( ] │ │ { │ push '{' │ [ (, { ] │ │ [ │ push '[' │ [ (, {, [ ] │ │ ] │ pop '[' │ [ (, { ] ✓ 매칭 │ │ } │ pop '{' │ [ ( ] ✓ 매칭 │ │ ) │ pop '(' │ [ ] ✓ 매칭 │ └────────────────────────────────────────┘ stack 비어있음 → True ✅
📝 기본 풀이 코드 (Python)
class Solution: def isValid(self, s: str) -> bool: stack = [] mapping = {')': '(', '}': '{', ']': '['} for char in s: if char in mapping: top = stack.pop() if stack else '#' if mapping[char] != top: return False else: stack.append(char) return not stack
⚡ 최적화 버전 (Python)
class Solution: def isValid(self, s: str) -> bool: # 길이가 홀수면 즉시 False if len(s) % 2: return False stack = [] pairs = {'(':')', '{':'}', '[':']'} for c in s: if c in pairs: stack.append(pairs[c]) # 기대 닫는 괄호 저장 elif not stack or stack.pop() != c: return False return not stack
📊 복잡도 분석
구분복잡도이유
시간 (기본)O(n)문자열 한 번 순회
공간 (기본)O(n)최악의 경우 stack에 n/2개 저장
시간 (최적)O(n)동일하게 한 번 순회
공간 (최적)O(n)기대값 push로 동일 공간, 비교 1회 감소
LeetCode #155 — Min Stack Medium by LeetCode
🎨 알고리즘 시각화
연산: push(-2), push(0), push(-3), getMin(), pop(), getMin() main_stack: min_stack: ┌────┐ ┌────┐ │ -3 │ ←top │ -3 │ ← 현재 최솟값 │ 0 │ │ -2 │ │ -2 │ │ -2 │ └────┘ └────┘ getMin() = -3 pop() 후: ┌────┐ ┌────┐ │ 0 │ ←top │ -2 │ ← 복원된 최솟값 │ -2 │ │ -2 │ └────┘ └────┘ getMin() = -2 ✅
📝 기본 풀이 코드 (Python)
class MinStack: def __init__(self): self.stack = [] self.min_stack = [] def push(self, val: int) -> None: self.stack.append(val) if not self.min_stack or val <= self.min_stack[-1]: self.min_stack.append(val) def pop(self) -> None: top = self.stack.pop() if top == self.min_stack[-1]: self.min_stack.pop() def top(self) -> int: return self.stack[-1] def getMin(self) -> int: return self.min_stack[-1]
⚡ 최적화 버전 (Python)
class MinStack: # (값, 현재_최솟값) 쌍을 단일 스택에 저장 def __init__(self): self.stack = [] # (val, cur_min) def push(self, val: int) -> None: cur_min = min(val, self.stack[-1][1]) if self.stack else val self.stack.append((val, cur_min)) def pop(self) -> None: self.stack.pop() def top(self) -> int: return self.stack[-1][0] def getMin(self) -> int: return self.stack[-1][1]
📊 복잡도 분석
구분복잡도이유
시간 (모든 연산)O(1)push/pop/top/getMin 모두 상수 시간
공간 (기본)O(n)보조 min_stack 추가 공간 필요
공간 (최적)O(n)튜플 쌍 저장으로 스택 1개로 통합
LeetCode #84 — Largest Rectangle in Histogram Hard by LeetCode
🎨 알고리즘 시각화
heights = [2, 1, 5, 6, 2, 3] 6 │ ██ 5 │ ████ 3 │ ████ ██ 2 │██ ████ ████ 1 │████████████████ └───────────────── 0 1 2 3 4 5 Monotonic Stack 진행: index=2(h=5): stack=[1,2] → 5 확장 탐색 index=3(h=6): stack=[1,2,3] index=4(h=2): pop 6 → area=6*1=6 pop 5 → area=5*2=10 ← MAX push 4 index=5(h=3): stack=[1,4,5] 최종 정리: area=10 ✅
📝 기본 풀이 코드 (Python)
class Solution: def largestRectangleArea(self, heights: list[int]) -> int: stack = [] # (index, height) max_area = 0 for i, h in enumerate(heights): start = i while stack and stack[-1][1] > h: idx, height = stack.pop() max_area = max(max_area, height * (i - idx)) start = idx stack.append((start, h)) for i, h in stack: max_area = max(max_area, h * (len(heights) - i)) return max_area
⚡ 최적화 버전 (Python)
class Solution: def largestRectangleArea(self, heights: list[int]) -> int: # 경계값 0 추가로 나머지 처리 통합 heights = [0] + heights + [0] stack = [0] max_area = 0 for i in range(1, len(heights)): while heights[i] < heights[stack[-1]]: h = heights[stack.pop()] w = i - stack[-1] - 1 max_area = max(max_area, h * w) stack.append(i) return max_area
📊 복잡도 분석
구분복잡도이유
시간 (기본)O(n)각 원소는 push/pop 각 1회씩만 처리
공간 (기본)O(n)monotonic stack 최대 n개 저장
시간 (최적)O(n)sentinel 0 추가로 루프 1개로 통합
공간 (최적)O(n)동일, 인덱스만 저장해 튜플 오버헤드 제거
