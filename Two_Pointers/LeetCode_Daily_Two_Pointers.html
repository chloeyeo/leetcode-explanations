LeetCode #283 - Move Zeroes Easy

by LeetCode

Algorithm Visualization
nums = [0, 1, 0, 3, 12] Two pointers: slow=0 (insertion pos), fast scans fast=0: nums[0]=0 skip fast=1: nums[1]=1 nonzero -> swap(slow=0, fast=1) -> [1,0,0,3,12], slow=1 fast=2: nums[2]=0 skip fast=3: nums[3]=3 nonzero -> swap(slow=1, fast=3) -> [1,3,0,0,12], slow=2 fast=4: nums[4]=12 nonzero -> swap(slow=2, fast=4) -> [1,3,12,0,0], slow=3 Result: [1, 3, 12, 0, 0] âś“ relative order preserved
Basic Solution
class Solution: def moveZeroes(self, nums): slow = 0 for fast in range(len(nums)): if nums[fast] != 0: nums[slow], nums[fast] = nums[fast], nums[slow] slow += 1
Optimized Solution
class Solution: def moveZeroes(self, nums): insert = 0 for n in nums: if n: nums[insert] = n insert += 1 # Fill rest with zeros nums[insert:] = [0] * (len(nums) - insert)
Complexity Analysis
SolutionTimeSpaceReason
BasicO(n)O(1)Single pass; swap in-place
OptimizedO(n)O(1)Single pass; overwrite then fill zeros â€” fewer swaps
LeetCode #15 - 3Sum Medium

by LeetCode

Algorithm Visualization
nums = [-4,-1,-1,0,1,2] (sorted) Fix i=0 (-4): L=1(-1), R=5(2) sum=-3 < 0 -> L++ L=2(-1), R=5(2): sum=-3 < 0 -> L++ L=3(0), R=5(2): sum=-2 < 0 -> L++ L=4(1), R=5(2): sum=-1 < 0 -> L++ L==R -> stop Fix i=1 (-1): L=2(-1), R=5(2) sum=0 == 0 -> FOUND [-1,-1,2], L++, R-- L=3(0), R=4(1): sum=0 == 0 -> FOUND [-1,0,1], L++, R-- L==R -> stop Fix i=2: nums[2]==nums[1] -> SKIP (dedup) Fix i=3 (0): L=4(1), R=5(2) sum=3 > 0 -> R-- L==R -> stop Result: [[-1,-1,2],[-1,0,1]]
Basic Solution
class Solution: def threeSum(self, nums): nums.sort() res = [] for i in range(len(nums) - 2): if i > 0 and nums[i] == nums[i - 1]: continue L, R = i + 1, len(nums) - 1 while L < R: s = nums[i] + nums[L] + nums[R] if s == 0: res.append([nums[i], nums[L], nums[R]]) while L < R and nums[L] == nums[L + 1]: L += 1 while L < R and nums[R] == nums[R - 1]: R -= 1 L += 1; R -= 1 elif s < 0: L += 1 else: R -= 1 return res
Optimized Solution
class Solution: def threeSum(self, nums): nums.sort() res = [] for i, v in enumerate(nums): if v > 0: break # sorted: no triplet sums to 0 if pivot > 0 if i and v == nums[i - 1]: continue L, R = i + 1, len(nums) - 1 while L < R: s = v + nums[L] + nums[R] if s > 0: R -= 1 elif s < 0: L += 1 else: res.append([v, nums[L], nums[R]]) L += 1; R -= 1 while L < R and nums[L] == nums[L - 1]: L += 1 return res
Complexity Analysis
SolutionTimeSpaceReason
BasicO(n²)O(1)Sort O(n log n) + O(n) outer * O(n) inner two-pointer
OptimizedO(n²)O(1)Early break when pivot > 0 prunes significant portion
LeetCode #4 - Median of Two Sorted Arrays Hard

by LeetCode

Algorithm Visualization
nums1=[1,3] nums2=[2] -> total=3, median=2.0 Binary search on the SMALLER array (nums1): Partition nums1 at mid1=1, nums2 at mid2=1: nums1: [1] | [3] nums2: [2] | [] left side: max(1,2)=2 right side: min(3,inf)=3 left_max > right_min? 2 > 3? NO -> valid partition! median = (2+3)/2 = 2.5? No, total odd -> median = right_min = min(3,inf) = 2? Correct: mid1=1 means 1 element from nums1 on left left_max1=nums1[0]=1, left_max2=nums2[0]=2 right_min1=nums1[1]=3, right_min2=inf 1 <= inf AND 2 <= 3 -> valid total=3 odd -> median = max(1,2) = 2 âś“
Basic Solution
class Solution: def findMedianSortedArrays(self, nums1, nums2): if len(nums1) > len(nums2): nums1, nums2 = nums2, nums1 # ensure nums1 is smaller m, n = len(nums1), len(nums2) lo, hi = 0, m while lo <= hi: mid1 = (lo + hi) // 2 mid2 = (m + n + 1) // 2 - mid1 L1 = nums1[mid1 - 1] if mid1 > 0 else float('-inf') R1 = nums1[mid1] if mid1 < m else float('inf') L2 = nums2[mid2 - 1] if mid2 > 0 else float('-inf') R2 = nums2[mid2] if mid2 < n else float('inf') if L1 <= R2 and L2 <= R1: if (m + n) % 2 == 0: return (max(L1, L2) + min(R1, R2)) / 2 return max(L1, L2) elif L1 > R2: hi = mid1 - 1 else: lo = mid1 + 1
Optimized Solution
class Solution: def findMedianSortedArrays(self, nums1, nums2): A, B = nums1, nums2 if len(A) > len(B): A, B = B, A m, n, half = len(A), len(B), (len(A) + len(B)) // 2 lo, hi = 0, m while True: i = (lo + hi) // 2 j = half - i Aleft = A[i - 1] if i else float('-inf') Aright = A[i] if i < m else float('inf') Bleft = B[j - 1] if j else float('-inf') Bright = B[j] if j < n else float('inf') if Aleft <= Bright and Bleft <= Aright: if (m + n) % 2: return min(Aright, Bright) return (max(Aleft, Bleft) + min(Aright, Bright)) / 2 elif Aleft > Bright: hi = i - 1 else: lo = i + 1
Complexity Analysis
SolutionTimeSpaceReason
BasicO(log min(m,n))O(1)Binary search on smaller array only
OptimizedO(log min(m,n))O(1)Cleaner partition logic; A/B naming improves clarity
