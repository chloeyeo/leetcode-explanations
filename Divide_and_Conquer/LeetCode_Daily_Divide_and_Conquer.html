#169 Majority Element — by LeetCode
Divide and Conquer · Boyer-Moore Voting
Algorithm Visualization
Divide and Conquer: Majority Element Input: [3,2,3,1,2,3,3] DIVIDE: [3,2,3,1,2,3,3] / [3,2,3] [1,2,3,3] / \ / [3] [2,3] [1,2] [3,3] CONQUER (return majority of each half): [2] vs [3] -> 3 wins [1] vs [2] -> tie, pick left=1 (irrelevant) [3] vs [3] -> 3 [3,2,3]: count(3)=2 > count(2)=1 -> 3 [1,2,3,3]: count(3)=2 > count(1)=1 -> 3 [3] vs [3] -> 3 Final result: 3 (appears > n/2 times)
Basic Solution
def majorityElement(self, nums): # D&C: recursively find majority in each half def majority(lo, hi): if lo == hi: return nums[lo] mid = (lo + hi) // 2 left = majority(lo, mid) right = majority(mid + 1, hi) if left == right: return left lc = sum(1 for i in range(lo, hi+1) if nums[i]==left) rc = sum(1 for i in range(lo, hi+1) if nums[i]==right) return left if lc > rc else right return majority(0, len(nums)-1)
Optimized Solution (Boyer-Moore)
def majorityElement(self, nums): # Boyer-Moore Voting: O(n) time, O(1) space candidate, count = None, 0 for num in nums: if count == 0: candidate = num count += 1 if num == candidate else -1 return candidate
Complexity Analysis

#23 Merge K Sorted Lists — by LeetCode
Divide and Conquer · Priority Queue (Min-Heap)
Algorithm Visualization
Divide and Conquer: Merge K Lists Lists: [1->4->5], [1->3->4], [2->6] D&C Merge Tree: lists[0..2] / lists[0..1] lists[2] | | merge(L0,L1) [2->6] = 1->1->3->4->4->5 Final merge: 1->1->3->4->4->5 vs 2->6 1,1,2,2,3,4,4,5,6 -> 1->1->2->3->4->4->5->6 Min-Heap approach: Heap: [(1,0,n), (1,1,n), (2,2,n)] Pop (1,0) -> push (4,0) Pop (1,1) -> push (3,1) Pop (2,2) -> push (6,2) ... until heap empty
Basic Solution (D&C)
def mergeKLists(self, lists): if not lists: return None if len(lists) == 1: return lists[0] mid = len(lists) // 2 left = self.mergeKLists(lists[:mid]) right = self.mergeKLists(lists[mid:]) return self._merge(left, right) def _merge(self, l1, l2): dummy = cur = ListNode(0) while l1 and l2: if l1.val <= l2.val: cur.next = l1; l1 = l1.next else: cur.next = l2; l2 = l2.next cur = cur.next cur.next = l1 or l2 return dummy.next
Optimized Solution (Min-Heap)
import heapq def mergeKLists(self, lists): # O(n log k) time, O(k) heap space heap = [] for i, node in enumerate(lists): if node: heapq.heappush(heap, (node.val, i, node)) dummy = cur = ListNode(0) while heap: val, i, node = heapq.heappop(heap) cur.next = node; cur = cur.next if node.next: heapq.heappush(heap, (node.next.val, i, node.next)) return dummy.next
