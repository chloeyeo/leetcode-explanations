LeetCode #70 — Climbing Stairs
Easy by LeetCode
📊 Algorithm Visualization

🐍 Basic Solution (Python)
def climbStairs(self, n: int) -> int:
    if n <= 2:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2
    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
⚡ Optimized Solution — O(1) Space (Python)
def climbStairs(self, n: int) -> int:
    if n <= 2:
        return n
    prev2, prev1 = 1, 2
    for _ in range(3, n + 1):
        curr = prev1 + prev2     # only need last 2
        prev2, prev1 = prev1, curr
    return prev1
📈 Complexity Analysis
VersionTimeSpaceReason

OptimizedO(n)O(1)Loop n times; only 2 variables kept
LeetCode #322 — Coin Change
Medium by LeetCode
📊 Algorithm Visualization
coins=[1,3,4], amount=6 dp[0]=0 (base case: 0 coins needed for amount 0) amount: 0 1 2 3 4 5 6 dp: 0 1 2 1 1 2 2 ← answer! For dp[5]: coin=1: dp[5-1]+1 = dp[4]+1 = 2 coin=3: dp[5-3]+1 = dp[2]+1 = 3 coin=4: dp[5-4]+1 = dp[1]+1 = 2 min = 2 Recurrence: dp[i] = min(dp[i - c] + 1) for c in coins
🐍 Basic Solution (Python)
def coinChange(self, coins: list[int], amount: int) -> int:
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for i in range(1, amount + 1):
        for coin in coins:
            if i - coin >= 0:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1
⚡ Optimized Solution — Early Exit + Sorted Coins (Python)
def coinChange(self, coins: list[int], amount: int) -> int:
    dp = [amount + 1] * (amount + 1)  # use amount+1 as sentinel
    dp[0] = 0
    coins.sort()  # sort to enable early break
    for i in range(1, amount + 1):
        for coin in coins:
            if coin > i:
                break  # no need to check larger coins
            dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] <= amount else -1
📈 Complexity Analysis
VersionTimeSpaceReason
BasicO(amount × |coins|)O(amount)Nested loop over all amounts and coins; dp array
OptimizedO(amount × |coins|)O(amount)Same asymptotic but early break reduces constant factor
LeetCode #124 — Binary Tree Maximum Path Sum
Hard by LeetCode
📊 Algorithm Visualization
Tree: -10 / \ 9 20 / \ 15 7 DFS post-order: node 9 → gain = max(9, 0) = 9 node 15 → gain = max(15, 0) = 15 node 7 → gain = max(7, 0) = 7 node 20: left_gain = 15 right_gain = 7 path_through_20 = 20 + 15 + 7 = 42 → new global max! return 20 + max(15, 7) = 35 (pass upward) node -10: left_gain = max(9, 0) = 9 right_gain = max(35, 0) = 35 path_through_-10 = -10 + 9 + 35 = 34 global max stays 42 Answer: 42
🐍 Basic Solution (Python)
def maxPathSum(self, root: TreeNode) -> int:
    self.max_sum = float('-inf')

    def dfs(node):
        if not node:
            return 0
        left  = max(dfs(node.left),  0)   # ignore negative
        right = max(dfs(node.right), 0)
        self.max_sum = max(self.max_sum,
                          node.val + left + right)
        return node.val + max(left, right)  # best single branch

    dfs(root)
    return self.max_sum
⚡ Optimized Solution — No Self Attribute (Python)
def maxPathSum(self, root: TreeNode) -> int:
    max_sum = [float('-inf')]  # mutable container

    def dfs(node) -> int:
        if not node:
            return 0
        l = max(dfs(node.left),  0)
        r = max(dfs(node.right), 0)
        max_sum[0] = max(max_sum[0], node.val + l + r)
        return node.val + max(l, r)

    dfs(root)
    return max_sum[0]
📈 Complexity Analysis
VersionTimeSpaceReason
BasicO(n)O(h)Visit each node once; recursion stack = tree height h
OptimizedO(n)O(h)Same; avoids self attribute access overhead per call
