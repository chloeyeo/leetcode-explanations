#997 Find the Town Judge Easy by LeetCode
Algorithm Visualization
Town: n=4, trust=[[1,3],[1,4],[2,3],[2,4],[4,3]] Person: 1 2 3 4 | | /| / +---------+ / down --- in-degree: [0, 0, 3, 2] (how many trust them) out-degree: [0, 2, 0, 1] (how many they trust) Judge: in-degree == n-1 AND out-degree == 0 Person 3: in=3 (=n-1=3) OK, out=0 OK => JUDGE = 3
Basic Solution
class Solution: def findJudge(self, n: int, trust) -> int: in_deg = [0] * (n + 1) out_deg = [0] * (n + 1) for a, b in trust: out_deg[a] += 1 in_deg[b] += 1 for i in range(1, n + 1): if in_deg[i] == n - 1 and out_deg[i] == 0: return i return -1
Optimized Solution (Single Score Array)
class Solution: def findJudge(self, n: int, trust) -> int: # score[i] = in_degree - out_degree # Judge has score == n-1 score = [0] * (n + 1) for a, b in trust: score[a] -= 1 score[b] += 1 for i in range(1, n + 1): if score[i] == n - 1: return i return -1
Complexity Analysis
VersionTimeSpaceReason
BasicO(E+N)O(N)Iterate trust edges once + scan N persons; 2 arrays
OptimizedO(E+N)O(N)Same but single array — 2x better constant
#207 Course Schedule Medium by LeetCode
Algorithm Visualization
numCourses=4, prerequisites=[[1,0],[2,0],[3,1],[3,2]] Graph: 0 --> 1 --> 3 0 --> 2 --> 3 DFS 3-state cycle detection: state: 0=unvisited 1=visiting(in stack) 2=done Visit 0: state[0]=1 Visit 1: state[1]=1 Visit 3: state[3]=1 => no neighbors => state[3]=2 OK state[1]=2 OK Visit 2: state[2]=1 Visit 3: state[3]==2 already done => skip OK state[2]=2 OK state[0]=2 OK => No cycle => can finish!
Basic Solution (DFS)
class Solution: def canFinish(self, numCourses: int, prerequisites) -> bool: from collections import defaultdict graph = defaultdict(list) for a, b in prerequisites: graph[b].append(a) state = [0] * numCourses def dfs(node): if state[node] == 1: return False # cycle if state[node] == 2: return True # already OK state[node] = 1 for nei in graph[node]: if not dfs(nei): return False state[node] = 2 return True return all(dfs(i) for i in range(numCourses) if state[i] == 0)
Optimized Solution (Kahn BFS / Topological Sort)
class Solution: def canFinish(self, numCourses: int, prerequisites) -> bool: from collections import defaultdict, deque in_degree = [0] * numCourses graph = defaultdict(list) for a, b in prerequisites: graph[b].append(a); in_degree[a] += 1 q = deque(i for i in range(numCourses) if in_degree[i] == 0) done = 0 while q: node = q.popleft(); done += 1 for nei in graph[node]: in_degree[nei] -= 1 if in_degree[nei] == 0: q.append(nei) return done == numCourses
Complexity Analysis
VersionTimeSpaceReason
DFSO(V+E)O(V+E)Visit each node/edge once; recursion stack O(V)
BFS KahnO(V+E)O(V+E)Queue processes each node once; no recursion risk
#332 Reconstruct Itinerary Hard by LeetCode
Algorithm Visualization
tickets=[["JFK","MUC"],["MUC","LHR"],["LHR","SFO"],["SFO","SJC"]] Adjacency (sorted desc for pop): JFK -> [MUC] MUC -> [LHR] LHR -> [SFO] SFO -> [SJC] Hierholzer iterative (Eulerian Path): stack=[JFK] JFK has MUC => push MUC stack=[JFK,MUC] MUC has LHR => push LHR stack=[JFK,MUC,LHR] LHR has SFO => push SFO stack=[JFK,MUC,LHR,SFO] SFO has SJC => push SJC stack=[JFK,MUC,LHR,SFO,SJC] SJC no neighbors => pop => result=[SJC] ...backtrack all => result=[SJC,SFO,LHR,MUC,JFK] Reverse => [JFK, MUC, LHR, SFO, SJC] OK
Basic Solution (Recursive DFS)
class Solution: def findItinerary(self, tickets) -> List[str]: from collections import defaultdict graph = defaultdict(list) for s, d in sorted(tickets, reverse=True): graph[s].append(d) res = [] def dfs(ap): while graph[ap]: dfs(graph[ap].pop()) res.append(ap) dfs("JFK") return res[::-1]
Optimized Solution (Hierholzer Iterative)
class Solution: def findItinerary(self, tickets) -> List[str]: from collections import defaultdict graph = defaultdict(list) for s, d in sorted(tickets, reverse=True): graph[s].append(d) stack, res = ["JFK"], [] while stack: while graph[stack[-1]]: stack.append(graph[stack[-1]].pop()) res.append(stack.pop()) return res[::-1]
Complexity Analysis
VersionTimeSpaceReason
RecursiveO(E log E)O(E)Sort dominates; DFS visits each edge once; recursion depth O(E)
IterativeO(E log E)O(E)Same sort; iterative avoids Python recursion limit for large inputs
